## 1.2 F1（CRITICAL）— S4 的「复用 EditorSimulateBuildInvoker 换参数」不成立

**设计原文（§4 / §5 S4 / §11）：**
> "调用 YooAsset **生产构建**（与现有 `EditorSimulateBuildInvoker.Build` 同源，但走 Builtin/Scriptable Build Pipeline 而非 VirtualRawBundle 模拟）"

**代码事实：**
- `KJHybridClrBuildTools.cs` 中唯一的 YooAsset 构建调用是：
  ```csharp
  var result = EditorSimulateBuildInvoker.Build(packageName, (int)EBundleType.VirtualRawBundle);
  ```
- `EditorSimulateBuildInvoker` 与 `EBundleType` 均来自 **YooAsset 包自身**（全工程 `Assets/` 内无其定义），且参数 `VirtualRawBundle` 是**模拟专用虚拟包**。类名、参数语义都表明它是 simulate-only 的编辑器构建器。

**影响：** S4 不是「复用 + 换参」，而是**新接 YooAsset 生产构建 API** 的集成工作。设计把它降级成「风险表一项」，低估了工作量，且 M2 里程碑（「实现生产构建调用」）在 API 未确认前不应排期。

**修正建议：**
- S4 = 驱动 YooAsset 真实生产构建（具体 API 取决于当前 YooAsset 版本，需先确认：`AssetBundleBuilder.BuildPackage` 还是 YooAsset 构建管线资产里选定的 pipeline）。
- 在 M2 之前先做一个 **S4 原型**：用最小工程验证能产出 `DefaultPackage` 真包并落到正确位置（见 F4）。
- 设计 §11 的「复用」列表应把 `EditorSimulateBuildInvoker` 从「同源复用」改为「不可复用，需另接生产构建 API」，并新增「确认 YooAsset 生产构建 API」作为 M2 的前置调研任务。

## 1.3 F2（CRITICAL）— S5 改错了配置字段，且遗漏 AssetConfig.Mode 切换与回滚

**设计原文（§5 S5 / §7 BuildConfig）：**
> "根据 `BuildConfig` 设置 `streamingAssetsRoot`（离线冒烟走 builtin，故指向 local StreamingAssets）、`skipHotUpdateInEditor = false`"

**代码事实（`AssetConfig.cs`）：**
```csharp
public PlayMode Mode = PlayMode.EditorSimulate;   // 默认就是 EditorSimulate
public string PackageName = "DefaultPackage";
public string EditorSimulatePackageRoot;
public string CdnBaseUrl = "http://127.0.0.1:8080/CDN";
public enum PlayMode { EditorSimulate, Offline, Host }   // Offline/Host 才是 Player 用的
```
- `AssetConfig` **没有 `streamingAssetsRoot` 字段**。运行时开关在 `BootLoader.BuildOptions`（`BootLoader.cs:97-118`）按 `config.Mode` 分支：`EditorSimulate` → 模拟；`Offline` → `BuiltinFileSystem`（读 StreamingAssets）；`Host` → builtin + cache（走 CDN）。
- 也就是说，Player 离线冒烟要能跑，必须把 **`AssetConfig.Mode` 设为 `Offline`**（不是设置某个 root 字段）。**默认值 `EditorSimulate` 在 Player 里会直接失败**（模拟模式只在编辑器内有效）。

**影响：** 设计 S5 写的字段根本不存在，且没提 `Mode` 切换。按设计实现，Player 启动会因 `EditorSimulate` 模式在 YooAsset 初始化阶段就挂掉，S8 必然失败。

**还有一个被忽略的点——回滚：** `AssetConfig` 是共享 ScriptableObject，开发者的 Editor Play 循环依赖 `Mode = EditorSimulate`。管线为了构建把它改成 `Offline` 后，**必须构建完回滚成 `EditorSimulate`**，否则开发者下次 Editor Play 会炸。设计完全没有提回滚/或「用独立的构建用 AssetConfig」的方案。

**修正建议：**
- S5 应：加载 `AssetConfig` → 设 `config.Mode = AssetConfig.PlayMode.Offline`、`config.PackageName = "DefaultPackage"` → `EditorUtility.SetDirty(config)` + `AssetDatabase.SaveAssets()` + `AssetDatabase.Refresh()`。
- S6 之后新增一步：把 `config.Mode` 回滚为 `EditorSimulate` 并保存（或方案 B：构建期间临时替换为一份 build-only 的 AssetConfig 资产，不污染主资产）。
- `BootStartupSettings`（Entry 上的）确实有 `EnableHotUpdate` / `SkipHotUpdateInEditor`，这些字段设计提对了，保留；但 `streamingAssetsRoot` 应删除，改为 `AssetConfig.Mode`。

## 1.4 F3（HIGH）— S8 读错日志文件，会漏掉最可能的启动期失败

**设计原文（§5 S8 / §8）：**
> "启动 Player → 抓 `Logs/Runtime/latest.jsonl` → 判定 Boot 到 ProjectStartup 成功"

**代码事实：**
- `BootStartupLog.cs:65-72`：AOT 阶段日志写到 **`Logs/Runtime/boot.log`**（纯文本），路径取 `Application.persistentDataPath/Logs/Runtime`（Player 下）或 projectRoot（编辑器下）。它**不写 `latest.jsonl`**。
- `latest.jsonl` / `latest.session.json` 由 `RuntimeLogSession.cs` 写入，而 `RuntimeLogSession` 是**热更层 RuntimeLog** 起来后才安装的（由 `BootUpdateRunner.ReplayEarlyLogs` 之后的 RuntimeLog 链路创建）。
- 因此：
  - **AOT 阶段（`BootLoader`：YooAsset 初始化、加载热更 DLL、Assembly.Load）的成败只出现在 `boot.log`**。
  - **`[Boot] Starting game` / `ProjectStartup` / `ProjectLifetimeScope` 里程碑出现在 `latest.jsonl`**（RuntimeLog 起来后）。

**影响：** 第一次真构建最可能在 AOT 阶段（YooAsset 初始化失败、DLL 加载/元数据不全）就挂。若 S8 只盯 `latest.jsonl`，会读到「空」或「根本没创建」，误判为「无证据/超时」，而真正的错误在 `boot.log` 里。

**修正建议：**
- S8 同时读取 **`boot.log` 与 `latest.jsonl`**（二者同目录 `<persistentDataPath>/Logs/Runtime/`）。
- 成功判定 = `boot.log` 无 AOT 阶段错误 **且** `latest.jsonl` 含 `[Boot] Starting game` / `ProjectStartup.Start` / `ProjectLifetimeScope` 里程碑。
- 设计必须**明确日志路径来源**：Player 下是 `Application.persistentDataPath/Logs/Runtime`（Win 即 `%USERPROFILE%/AppData/LocalLow/<company>/<product>/Logs/Runtime`）。冒烟脚本需按平台解析该路径，不能直接假设项目根下的 `Logs/Runtime`。

## 1.5 F4（HIGH）— S4 输出位置与 Offline builtin 读取位置不匹配

**设计原文（§5 S4）：**
> "输出根 = `Bundles/<Platform>/DefaultPackage/`"

**代码事实：** `BootLoader` 的 `Offline` 模式用 `BuiltinFileSystemParameters = FileSystemParameters.CreateDefaultBuiltinFileSystemParameters(packageName)`。YooAsset builtin 文件系统默认从 **`StreamingAssets/{packageName}`**（即 `StreamingAssets/DefaultPackage`）读取；而 `BuildPlayer` 会把工程的 `Assets/StreamingAssets/` 整体打进 Player。

**影响：** 若 S4 只把真包输出到 `Bundles/<Platform>/DefaultPackage/` 而不放进 `Assets/StreamingAssets/DefaultPackage`，OffLine 模式的 builtin 文件系统在 Player 里找不到包 → YooAsset 初始化失败（与 F2 叠加，必挂）。

**修正建议：**
- S4 对 **Offline** 目标，必须把真包落到 `Assets/StreamingAssets/DefaultPackage`（或确认 YooAsset builtin 构建默认就输出到该处，构建后再 `AssetDatabase.Refresh` 让 BuildPlayer 纳入）。
- `Bundles/<Platform>/` 更适合 **Host** 模式（配合 CDN）。离线冒烟优先，故 S4 的 Offline 分支以 StreamingAssets 为准。
- S7 静态校验应额外确认 `Assets/StreamingAssets/DefaultPackage`（或构建产物中）含 10+3 个 DLL，而非只看 `Bundles/` 目录。

# 第二部分 — 运行时 Asset 系统审查

> 整合自 `Hy3_Asset系统_代码审查_2026-07-08.md`。审查对象：`Assets/Framework/Asset/AssetRuntime.cs`、`AssetHandle.cs`、`AssetInstanceHandle.cs`，以及 YooAsset 运行时源码（`Library/PackageCache/com.tuyoogame.yooasset@bfb166ad91/Runtime/`）。

## 2.0 YooAsset 内部事实（背景，合并去重）

- **无对象池**：全 `Runtime/` 目录搜 `Pool` / `ObjectPool` / `Stack<Handle>` / `Queue<Handle>` 零命中。`AsyncOperationSystem` 仅做按帧调度，不池化 Operation。
- **句柄不可复用**：`HandleBase.Release()`（`HandleBase.cs:28`）执行 `Provider.ReleaseHandle(this)` 后 `Provider = null`（`:38`）。此后 `IsValid` 为 `false`，`CheckValidWithWarning()` 直接打 `"Handle has already been released"` 并短路。整个 `HandleBase` 无任何 Reset / 重新绑定 API——**释放即死亡**。
- **复用在 Provider 层**：`ResourceManager._providerDict`（`ResourceManager.cs:18`）按 `providerKey = nameof(LoadAssetAsync) + assetInfo.AssetKey` 缓存 Provider（`:215-221`）。重复加载同一资源返回同一 Provider，`ProviderBase.CreateHandle<T>()`（`:275`）仅 `RefCount++` 并 `new` 一个薄句柄。所有句柄 Release、`RefCount==0` 且开启 `AutoUnloadBundleWhenUnused` 时，`TryUnloadBundle()`（`:316`）才延迟卸载。

> **推论（贯穿全文）：** 业务层**不该**在句柄上做对象池；真正要管的是「句柄生命周期 / 引用计数」，以及「要池化的是 GameObject 实例（`Framework.Pool`）而非 asset handle」。复用在 YooAsset 的 Provider 层自动完成。

## 2.1 A1（CRITICAL）— `LoadAssetAsync<T>` 返回无主借用，可被任意 `Release` 释放掉

**代码事实（`AssetRuntime.cs`）：**
```csharp
// LoadAssetAsync<T>(path) — 251 行附近
if (_assetHandles.TryGetValue(key, out var cached) && cached.AssetObject is T asset)
    return asset;          // 直接把资源对象还给调用方，不附带任何所有权
...
_assetHandles[key] = handle;            // 310 行：handle 被缓存
var loadedObject = shouldCache ? handle.AssetObject : null;  // 314 行：只返回对象
```
```csharp
// Release<T>(path) — 441 行 / Release(path) — 462 行
if (_assetHandles.TryGetValue(key, out handle)) {
    _assetHandles.Remove(key);
}
handle?.Release();          // 459 / 504 行：缓存被清，底层 YooAsset handle 被 Release
```

**问题本质：** 缓存中的 YooAsset handle 之所以「活着」，仅仅是因为「还没人 Release 它」。但 `LoadAssetAsync<T>` 把 `asset` 交给调用方后，**调用方手上没有任何能阻止它被释放的句柄**。一旦项目里另一处（甚至同一调用方后续逻辑）调了 `Release<T>(path)` 或 `Release(path)`，缓存被清、`handle.Release()`、`RefCount` 可能归零、`TryUnloadBundle` 卸载 AB —— 而第一个调用方仍持有 `asset` 引用。

**典型故障链：**
1. 系统 A：`var mat = LoadAssetAsync<Material>("M")`，长期持有 `mat`；
2. 系统 B：`Release("M")`（或 A 自己某处清理）；
3. AB 卸载，`mat` 变成失效对象；
4. A 继续使用 `mat` → 静默空引用 / 画面炸裂，且无报错堆栈指向根因。

**对比：** `LoadAssetHandleAsync<T>` + `AssetHandle<T>.Dispose()`（`:221` / `AssetHandle.cs:34`）是**单所有者 RAII 模型**——只有 `Dispose` 才释放，且 `Dispose` 回调从 `_ownedAssetHandles` 移除（`:248`）。这条路径是安全的；但 `LoadAssetAsync<T>` 完全绕开了它。

> 这是「共享可变缓存 + 无主借用」的经典别名（aliasing）陷阱，属于正确性级缺陷，不是优化项。**解法见第三部分 §3.2.1（owner 引用计数 + 自动剪空）。**

## 2.2 A2（HIGH）— 两套互不查询的簿记，导致重复句柄 + `InstantiateAsync` 绕过缓存

**代码事实：**
- `_assetHandles`：`Dictionary<AssetCacheKey, YooAsset.AssetHandle>`，按 `(path, type)` 缓存，**无人持有所有权**，仅显式 `Release` 才清（`:17`）。
- `_ownedAssetHandles`：`HashSet<YooAsset.AssetHandle>`，每个 entry 由一个 `AssetHandle<T>` 在 `Dispose` 时释放（`:20`、`:247`、`:248`）。

两条路径**从不互相查询**：
- `LoadAssetHandleAsync<T>`（`:221`）与 `InstantiateAsync`（`:339`）直接 `_defaultPackage.LoadAssetAsync<T>(path)`，并 `new AssetHandle<T>(handle, ...)` 加入 `_ownedAssetHandles`（`:247`）。**不查 `_assetHandles`**。
- 因此：若某资源已被 `LoadAssetAsync` 缓存（持有 handle H1，RefCount=1），再调 `InstantiateAsync` 同一路径会 `new` 出第二个 YooAsset handle H2，H1 与 H2 指向同一 Provider，`RefCount` 变 2，但两份簿记完全独立。
- **最讽刺的一点**：`InstantiateAsync`（游戏里最高频的调用）走的是 `LoadAssetHandleAsync` 路径，对 `_assetHandles` 缓存**完全视而不见**。那个「资源缓存」在实例化场景形同虚设。

**影响：** 同一资源可能同时存在两份 YooAsset 句柄、双份簿记；缓存层与拥有层分裂，既浪费又难推理，是后续埋雷的设计异味。**解法见第三部分 §3.2.3（单一账本）。**

## 2.3 A3（MEDIUM）— 无容量上限 / 无淘汰 / 无 LRU

**代码事实：** `_assetHandles`、`_ownedAssetHandles`、`_sceneHandles`（`:17-18`）均只增不减，唯一释放途径是显式 `Release` 或 `Shutdown`（`:89-124`）。没有任何容量上限、LRU、或基于时间的自动卸载。

**影响：** 长会话中若加载了大量不同资源却未逐个 `Release`，内存**单调增长**。对于热更新游戏（多系统、多界面反复进出）这是实打实的内存压力来源。**解法见第三部分 §3.2.4（每帧增量 + 内存压力分级淘汰）。**

## 2.4 A4（LOW）— 次要问题

### 2.4.1 锁约定不一致
`_assetHandles` / `_loadingTasks` 访问都包在 `lock(_gate)` 内；但 `LoadAssetHandleAsync` 中的 `_ownedAssetHandles.Add(handle)`（`:247`）与 `AssetHandle<T>` Dispose 回调里的 `Remove`，都**在锁外**执行。Unity 主线程模型下大概率无碍，但约定不统一，跨线程调用时存在竞态隐患。

### 2.4.2 `Release` 两个重载易用错
- `Release<T>(path)`（`:441`）：按精确 `(path, type)` key 释放；
- `Release(path)`（`:462`）：按 path 释放该路径下所有 type。

调用方若记错重载（例如 `LoadAssetAsync<GameObject>` 后调 `Release<Material>(path)`），会静默 no-op，导致资源漏释放，且与 A3 叠加放大内存增长。**解法见第三部分 §3.2.2（`AssetCachePolicy` 枚举消灭歧义 Release）。**

## 2.5 修正路线（运行时侧 P0–P3，与第三部分 §3.4 合并）

> 核心方向：**所有加载收敛到「拥有型句柄」模型，废弃/改造裸 `LoadAssetAsync<T>` 无主缓存路径，并给缓存加淘汰。**

| 优先级 | 问题 | 建议改法 |
|--------|------|----------|
| P0 | A1 无主借用 | `LoadAssetAsync<T>` 改为返回拥有型对象（或直接统一走 `AssetHandle<T>`），调用方 `Dispose` 才释放；若短期保留裸缓存，必须在文档/注释中明确「仅限短生命周期一次性使用」，并禁止对其调用 `Release`。 |
| P1 | A2 双份簿记 | `LoadAssetHandleAsync` / `InstantiateAsync` 先查统一账本复用同一 YooAsset handle；合并两套账本为统一的所有权管理。 |
| P1 | A3 无淘汰 | 给缓存增加容量上限 + 时间延迟 + LRU/压力淘汰，满了自动 `Release` 最久未用项。 |
| P2 | A4 锁/重载 | 统一加锁约定；收敛 `Release` 重载为避免歧义的单一 API（或让缓存层自管生命周期、不对外暴露 `Release`）。 |

## 3.0 核心结论 + KJ 现状

老系统最值得抄的不是「它怎么加载 AB」，而是**它怎么管住已加载资源的生命周期**：用 `CachedAsset<T>` 把「裸资源对象」包成**带 owner 引用计数、带内存代价、带常驻标记、带自动剪空 owner 安全网**的缓存项，再用 `AssetCacheMgr` 做**每帧增量、按内存压力分级、带时间延迟**的淘汰。这套机制精确命中我们 A1（无主借用）/ A2（双份簿记）/ A3（无限增长）/ A4（Release 歧义），且代码量极小（核心文件均 < 300 行），可直接移植。

> **注意 1（不替换 YooAsset）：** KJ 用 YooAsset 做底层加载，所以**不要替换 YooAsset**，而是把 `CachedAsset` 所有权模型**包在 YooAsset handle 外面**。YooAsset handle 仍由它自己管理引用计数与 AB 卸载；我们只在它之上加一层业务级所有权/淘汰。

## 3.2 可借鉴的设计思想（按价值排序，含代码）

### 3.2.1 Owner 引用计数 + 自动剪空安全网（解决 A1，最关键）

老系统的 `CachedAsset<T>` 不把资源裸交出去，而是让**每个持有者（owner，通常是 GameObject/Form/Node）显式 Retain/Release**：

```csharp
// CachedAsset.cs — 核心（完整版见附录 A.2）
private readonly List<Object> _weakRefList = new List<Object>();   // 谁持有我
public int WeakRefCount => _weakRefList.Count;

public void Retain(Object owner) {
    if (!IsValid) return;
    _clearTimestamp = Time.realtimeSinceStartup + CacheTime;
    if (owner == null) return;
    _weakRefList.Add(owner);
}

public void Release(Object owner) {
    _clearTimestamp = Time.realtimeSinceStartup + CacheTime;
    for (int i = _weakRefList.Count - 1; i >= 0; i--) {
        if (_weakRefList[i] == null) _weakRefList.RemoveAt(i);          // 剪掉已销毁的
        else if (_weakRefList[i] == owner) _weakRefList.RemoveAt(i);    // 剪掉指定 owner
    }
}

// 淘汰检查时，再次兜底剪空：即使调用方忘了 Release，owner 被 Destroy 后引用变 null，资源终可释放
public bool TickGC(int forceMemory = -1) {
    for (int i = _weakRefList.Count - 1; i >= 0; i--)
        if (_weakRefList[i] == null) _weakRefList.RemoveAt(i);
    if (Persistent) return false;
    if (_weakRefList.Count > 0) return false;
    if (forceMemory < 0 || Memory < forceMemory)
        return _clearTimestamp < Time.realtimeSinceStartup;
    return true;
}
```

**为什么这直接治本 A1**：我们现在的 `LoadAssetAsync<T>` 把 `asset` 裸返回、调用方不持有任何句柄，另一处 `Release` 就能把它卸载。老系统把「持有关系」显式建模进缓存项本身——资源知道自己被谁持有，且**即便忘了 Release，owner 被销毁（Unity 的 `== null`）也会在下次 TickGC 时被剪掉**，不会永久泄漏。这是比「调用方自觉 Release」稳得多的模型。

### 3.2.2 `AssetCachePolicy` 显式声明生命周期意图（解决 A1/A4）

```csharp
// AssetCachePolicy.cs（完整版见附录 A.5）
public enum AssetCachePolicy {
    ByOwner,    // 随 owner 存活，owner 销毁后释放（默认，推荐）
    Persistent, // 永久缓存
    DontCache,  // 不缓存
    Customize,  // 加载后由调用方手动设定

```

加载时把意图一并传入（`AssetLoadParameters.CacheOwner`），而不是事后用歧义的 `Release<T>`/`Release(path)` 去「撤销」。这同时消灭了我们的 A4（两个 Release 重载易用错）——调用方不再直接操控缓存释放，只需 `Retain/Release(owner)`。

### 3.2.3 单一账本 + 轮询增量扫描（解决 A2）

```csharp
// CachedAssetContainer.cs（完整版见附录 A.3）
private static readonly Dictionary<string, ICachedAsset> _assetMap;     // 名称 → 缓存项
private static readonly Dictionary<Object, ICachedAsset> _invertedDict; // 资源对象 → 缓存项（反向索引）
private static readonly List<string> _cachedNames;                       // 增量扫描游标
private static int _lastPos;                                             // 轮询位置

public static void CleanUnused(int step, int forceMemory = -1) {
    _clearList.Clear();
    int count = _cachedNames.Count;
    _lastPos = _lastPos >= count ? 0 : _lastPos;       // 环形轮询
    int end = _lastPos + step;
    for (int i = _lastPos; i < end && i < count; i++) {
        var cached = Find(_cachedNames[i]);
        if (cached != null && cached.TickGC(forceMemory))
            _clearList.Add(_cachedNames[i]);
    }
    _lastPos = end;
    foreach (var item in _clearList) Remove(item);    // Remove → UnloadAssetObject + RecycleToPool
}
```

**解决 A2**：我们目前 `_assetHandles`（裸缓存）与 `_ownedAssetHandles`（拥有型）两套互不查询的账本，导致同资源生成第二个 YooAsset handle、`InstantiateAsync` 绕过缓存。老系统只有**一个**账本，所有加载/实例化入口都走 `CachedAssetContainer.Add/Find`，天然杜绝重复 handle 与缓存分裂。`_invertedDict` 还能从「资源对象」反查缓存项，方便 `RetainAsset(asset, owner)` 这类按对象操作的 API。

### 3.2.4 每帧增量 + 内存压力分级淘汰（解决 A3）

```csharp
// AssetCacheMgr.cs（完整版见附录 A.4）
private const int MB = 1024 * 1024;
public static int CacheTime { get; set; } = 60;                 // 默认延迟 60s
private static long _memoryLimit = 300 * MB;                    // 300MB 上限
private static int[] _forceMemory = {MB, 2 * MB, 5 * MB};       // 压力分级强制清理阈值
private static int[] _memoryLvTemp = {30 * MB, 60 * MB, 120 * MB};

public static void Update() {
    int forceMemory = MemoryMonitor();          // 按 Profiler 总内存算压力等级
    if (forceMemory == -1) return;              // 没超阈值就不扫
    CleanStep(forceMemory);                      // 每帧只扫 _step(=10) 个，不卡帧
    AssetBundleCached.CleanAllUnused();
}
```

**解决 A3**：我们的三套字典只增不减，长会话内存单调增长。老系统用「时间延迟（默认 60s，给复用留窗口）+ 内存压力分级（越接近上限，越强制清理大资源）+ 每帧只扫 10 个（无卡顿）」三件套，把「无限增长」变成「有界、平滑、自适应」。

### 3.2.5 薄包装对象池（与 §2.0 结论一致）

`CachedAsset<T>` 自身进了 `ObjectPool`（复用 wrapper，不池资产）：
```csharp
internal static ObjectPool<CachedAsset<T>> Pool = ObjectPoolFactory.CreatePool<CachedAsset<T>>(64);
public void Reset() { /* 清状态 */ }
public void RecycleToPool() { Pool.Recycle(this); }
```
这印证了之前给的结论：**YooAsset 句柄不该池（它自己也没池），但包在它外面的轻量 wrapper 可以池**——既减少 GC，又不破坏 YooAsset 的引用计数模型。

### 3.2.6 资产缓存与实例池分层（强化 §2.0 备注）

老系统把「已加载资源」（`CachedAsset`，淘汰时重载）与「已实例化 GameObject」（`GameObjectResContainer`，回收复用）**分成两层**。这正好支持 §2.0 的结论：**要池的是 GameObject 实例（`Framework.Pool`），不是 asset handle**。KJ 已有 `Framework.Pool` / `Framework.Cache`，这套分层可直接对接。

### 3.2.7 可选：代价感知淘汰策略（A3 的进阶）

> **重要更正（已核对 KJ 现状）：** KJ 的 `Assets/Framework/Cache/` **已经自带** `LruCachePolicy.cs`（`LinkedList` O(1) LRU，与老系统 `LruCacheStrategy` 几乎一致）和 `ICacheEvictionPolicy<TKey>` 抽象接口。**所以 LRU 算法 KJ 早就有了，不需要从老系统搬。** 老系统真正额外有价值的淘汰算法是下面两个「代价感知」策略，可作为 KJ 现有 `ICacheEvictionPolicy` 的**补充策略**接入：

- **`PricedCachePool`**：每个 key 有 `price`（成本），超容量时**优先淘汰最便宜（最易重载/最小）**的资源——保留贵且难重载的。
- **`AccCostCachePool`**：按**累计成本**（频率×成本）淘汰，保留「高频且贵」的资源。
- （老系统另有 `FIFOCacheStrategy`，KJ 若需要 FIFO 也可补一个，逻辑同上。）

这些是老系统 `Package.Cache` 命名空间下的纯算法、不依赖 YooAsset，可低成本实现为 KJ `Framework.Cache` 的新 `ICacheEvictionPolicy` 实现（完整源码见附录 A.6 / A.7 / A.9 / A.10）。但**更关键的是「接进现有框架 + 补淘汰驱动器」**——KJ 有策略抽象却没把它驱动起来。

## 3.3 不值得照搬 / 需注意的地方

- **`ScriptsC#/General/Res/*` 是经济资源（金币/道具 tid+num），不是 Unity 资源加载**。`ResContainer`/`ResObj`/`ResNodeHolder` 那套是「背包/资源展示」，与我们 Asset 系统不是一回事，不要混淆。其 `ResFactory.GetResNode/RecycleResNode` 的「视图节点回收」思想可参考，但模型不同。
- 老 `Framework/UniCore/Asset` 是**自研 AB 加载器**（位于 YooAsset 之下）。KJ 用 YooAsset，所以 `AssetBundleManager.ReleaseAssetRef`、`AssetBundleIndex.GetMemory` 这些要**替换成 YooAsset 的等价物**（`handle.Release()`、以及自建一份资源内存估算表）。移植的是「所有权/淘汰」思想，不是 AB 加载实现。

## 4.1 老 → KJ 替换映射表（移植必读）

老系统代码耦合了一批老框架类型，移植到 KJ（YooAsset 底层）时必须按下表替换。**这是「一次成型」的关键前置**，否则直接抄源码会编译不过。

| `AssetBundleManager.ReleaseAssetRef(bundle, name)` | `yooHandle.Release()` | YooAsset handle 自管引用计数与 AB 卸载 |
| `AssetBundleIndex.GetMemory(name)` | KJ 自建资源内存估算表（按类型/大小粗略估算，或 `Profiler`/`Resources` 统计） | 老系统按 AB 估内存，KJ 需自估 |

**KJ 已有的 `Framework/Cache` 骨架（详见附录 B.1）：**
- `ICache<TKey,TValue>` / `ICacheResContainer<TKey,TValue>`：缓存容器接口（`TryGet`/`GetOrCreate`/`TryRemove`/`Clear`）。
- `ICacheEvictionPolicy<TKey>`：淘汰策略抽象（`Touch`/`Remove`/`TrySelectEvictionCandidate`/`Clear`）。

## A.2 `CachedAsset.cs`（所有权 + 淘汰核心，262 行）

```csharp
using System;
using System.Collections.Generic;
using Package.ObjectPool;
using UnityEngine;
using Object = UnityEngine.Object;

namespace UniCore
{
    /// <summary>
    /// 缓存资源对象
    /// </summary>
    public class CachedAsset<T> : ICachedAsset, IPooledObject where T : Object
    {
        //对象池
        internal static ObjectPool<CachedAsset<T>> Pool { get; } =
            ObjectPoolFactory.CreatePool<CachedAsset<T>>(64);

        //清理的延迟时间
        private static long CacheTime => AssetCacheMgr.CacheTime;

        //资源创建时间
        private float _createdTimestamp;

        //延迟清理时间（最后一次访问+延迟销毁时间）
        private float _clearTimestamp;

        //资源缓存

        ///资源内存
        public int Memory { get; private set; }

        /// <summary>
        /// 资源名称
        /// </summary>
        public string Name { get; private set; }

        /// <summary>
        /// 资源所在的AssetBundle名称
        /// </summary>
        public string AssetBundleName { get; private set; }

        /// <summary>
        /// 是否作为全局资源持久存在，不会在切换场景时释放
        /// </summary>
        public bool Persistent { get; set; }

        /// <summary>
        /// 这个资源是否为异步加载结果
        /// </summary>
        public bool IsAsyncLoaded { get; private set; }

        /// <summary>
        /// 加载的资源对象
        /// </summary>
        public T AssetObject { get; private set; }

        //引用该资源的对象列表
        private readonly List<Object> _weakRefList = new List<Object>();

        //引用数量
        public int WeakRefCount => _weakRefList.Count;

        /// <summary>
        /// IsValid为true，表示未被回收；否则表示已经回收Reset
        /// </summary>
        private bool IsValid => Name != null;

        /// <summary>
        /// 初始化函数
        /// </summary>
        /// <param name="parameters">资源加载参数</param>
        /// <param name="assetObject">加载到的资源对象</param>
        /// <param name="isAsyncLoaded">是否为异步加载结果</param>
        public void Init(AssetLoadParameters parameters, T assetObject, bool isAsyncLoaded)
        {
            Name = parameters.AssetName;
            if (string.IsNullOrEmpty(Name)) throw new ArgumentException("Asset name is null or empty.");

            AssetBundleName = parameters.BundleName;
            IsAsyncLoaded = isAsyncLoaded;
            AssetObject = assetObject;
            _createdTimestamp = Time.realtimeSinceStartup;
            _clearTimestamp = _createdTimestamp + CacheTime;
            Memory = AssetBundleIndex.GetMemory(Name);

            Retain(parameters.CacheOwner);
        }

        #region IPooledObject Implementation

        /// <summary>
        /// 重置所有状态，以便对象复用
        /// </summary>
        public void Reset()
        {
            Name = null;
            AssetBundleName = null;
            Persistent = false;
            _createdTimestamp = 0;
            _clearTimestamp = 0;
            AssetObject = null;
            _weakRefList.Clear();
            Memory = 0;
        }

        public IPool OriginPool
        {
            get => Pool;
            set { }
        }

        /// <summary>
        /// 放回对象池
        /// </summary>
        public void RecycleToPool()
        {
            Pool.Recycle(this);
        }

        #endregion

        /// <summary>
        /// 实现ICachedAsset接口
        /// </summary>
        /// <returns></returns>
        public Object GetAssetObject()
        {
            return AssetObject;
        }

        /// <summary>
        /// 释放加载的资源对象
        /// </summary>
        public void UnloadAssetObject()
        {
            if (string.IsNullOrEmpty(Name)) return;

            AssetLog.Debug("Asset.UnloadAssetObject() - " +
                           $"unloading name={Name} assetBundle={AssetBundleName} persistent={Persistent}");
            //反向索引中移除
            CachedAssetContainer.RemoveInvertedDict(AssetObject);
            AssetObject = null;

            //释放对应ab包
            AssetBundleManager.ReleaseAssetRef(AssetBundleName, Name);
        }

        /// <summary>
        /// 触发一次GC（回收资源）检查
        /// </summary>
        /// <param name="forceMemory">资源内存大于forceMemory，则跳过缓存时间</param>
        /// <returns>返回值表示此资源是否应该释放</returns>
        public bool TickGC(int forceMemory = -1)
        {
            //定时移除引用
            for (var i = _weakRefList.Count - 1; i >= 0; i--)
            {
                //如果引用被回收了，就移除改引用
                if (_weakRefList[i] == null)
                {
                    _weakRefList.RemoveAt(i);
                }
            }

            if (Persistent) return false;
            //如果还有引用
            if (_weakRefList.Count > 0) return false;

            //资源内存大于forceMemory，则跳过缓存时间
            if (forceMemory < 0 || Memory < forceMemory)
            {
                return _clearTimestamp < Time.realtimeSinceStartup;
            }

            AssetLog.Debug($"[{nameof(CachedAsset<T>)}]{nameof(TickGC)}: will clean asset, " +
                           $"name={Name} assetBundle={AssetBundleName}");

            return true;
        }

        public CachedAsset<T> Retain(AssetCacheOwner owner)
        {
            if (!IsValid) return null;

            _clearTimestamp = Time.realtimeSinceStartup + CacheTime;
            AssetLog.Debug($"{nameof(CachedAsset<T>)}.{nameof(Retain)} : asset:'{Name}', bundle:'{AssetBundleName}', " +
                           $"policy:{owner.Policy}");

            if (owner.IsPersistent)
            {
                Persistent = true;
            }
            else if (owner.Policy == AssetCachePolicy.ByOwner)
            {
#if UNITY_EDITOR
                if (owner.Owner == null)
                {
                    Debug.LogError($"owner.Owner({owner.Owner}) is null!");
                }
#endif
                _weakRefList.Add(owner.Owner);
            }

            return this;
        }


        public void Retain(Object owner)
        {
            if (!IsValid) return;

            _clearTimestamp = Time.realtimeSinceStartup + CacheTime;

            if (owner == null) return;
            _weakRefList.Add(owner);
        }

        public void Release(Object owner)
        {
            _clearTimestamp = Time.realtimeSinceStartup + CacheTime;
            for (var i = _weakRefList.Count - 1; i >= 0; i--)
            {
                // TODO 改为换位并移除最后一个，避免RemoveAt要整体移动数组
                //如果引用被回收了，就移除改引用
                if (_weakRefList[i] == null)
                {
                    _weakRefList.RemoveAt(i);
                }
                else if (_weakRefList[i] == owner)
                {
                    _weakRefList.RemoveAt(i);
                }
            }
        }

        /// <summary>
        /// 从oldOwner切换到newOwner
        /// </summary>
        /// <param name="oldOwner"></param>
        /// <param name="newOwner"></param>
        public void SwitchOwner(Object oldOwner, Object newOwner)
        {
            if (!IsValid) return;
            Release(oldOwner);
            Retain(newOwner);
        }

        public override string ToString()
        {
            if (WeakRefCount == 1)
            {
                return
                    $"wRef: {_weakRefList[0]} {_weakRefList[0].GetHashCode()}, Obj:{AssetObject} {AssetObject.GetHashCode()}";
            }
            else
            {
                return $"wRef_{WeakRefCount}, Obj:{AssetObject.GetHashCode()}";
            }
        }
    }
}
```

```csharp
        /// <param name="assetName">查找的资源名称</param>
        /// <returns>返回缓存资源对象</returns>
        public static CachedAsset<T> FindGeneric<T>(string assetName) where T : Object
        {
            if (_assetMap.TryGetValue(assetName, out var cache))
            {
                if (cache is CachedAsset<T> cacheT) return cacheT;
            }

            return null;
        }

        /// <summary>
        /// 通过资源查找缓存
        /// </summary>
        /// <param name="owner">资源对象</param>
        /// <return>缓存对象</return>
        public static ICachedAsset Find(Object owner)
        {
            if (owner == null)
            {
                Debug.LogError($"[{nameof(CachedAssetContainer)}.{nameof(Find)}] Owner is null");
                return null;
            }

            return _invertedDict.GetValueOrDefault(owner);
        }

        /// <summary>
        /// 添加一个资源到缓存中
        /// </summary>
        /// <param name="parameters">加载参数</param>
        /// <param name="assetObject">加载到的资源</param>
        /// <param name="isAsyncLoaded">是否为异步加载结果</param>
        public static CachedAsset<T> Add<T>(AssetLoadParameters parameters,
            T assetObject, bool isAsyncLoaded) where T : Object
        {
            var assetName = parameters.AssetName;

            CachedAsset<T> cacheT = null;

            // 先查看是否已经存在缓存
            if (_assetMap.TryGetValue(assetName, out var cache))
            {
                cacheT = cache as CachedAsset<T>;

                // 如果已经存在于缓存的资源类型与新增资源类型不一致，则报错，并返回null表示Add失败
                if (cacheT == null)
                {
                    Debug.LogError($"[{nameof(CachedAsset<T>)}.{nameof(Add)}] asset type not match," +
                                   $"existing cachedAsset is {cache.GetAssetObject()}, " +
                                   $"while new cachedAsset is {assetObject} ");
                    return null;
                }

                if (!cacheT.Persistent)
                {
                    cacheT.Persistent = parameters.CacheOwner.IsPersistent;
                }

                // 只有当同步和异步加载同时调用, 才可能出现重复Add
                // 所以如果两次Add都是同步, 或者都是异步, 则说明存在问题, 需要报错提示
                if (cacheT.IsAsyncLoaded == isAsyncLoaded)
                {
                    Debug.LogError($"[{nameof(CachedAsset<T>)}.{nameof(Add)}] cachedAsset already add by " +
                                   $"{(cache.IsAsyncLoaded ? "synchronized" : "async")} method, " +
                                   $"while current method is " + (isAsyncLoaded ? "synchronized" : "async"));
                }
            }
            else
            {

                // 加入缓存
                cacheT = CachedAsset<T>.Pool.Get();
                cacheT.Init(parameters, assetObject, isAsyncLoaded);
                _cachedNames.Add(assetName);
                _assetMap.Add(assetName, cacheT);
                _invertedDict[assetObject] = cacheT;
                AssetRecordMgr.Add(assetName);
            }

            return cacheT;
        }

                _cachedNames.Remove(assetName);
                asset.UnloadAssetObject();

            else
            {
                AssetLog.Info($"[{nameof(CachedAssetContainer)}.{nameof(Remove)}] " +
                              $"There is no cache for the asset:'{assetName}'");
            }
        }

        {

            _invertedDict.Remove(obj);

        /// <summary>
        /// 增量检查未使用资源
        /// </summary>
        /// <param name="step">检查步进</param>
        /// <param name="forceMemory">资源内存大于forceMemory，则跳过缓存时间</param>
        public static void CleanUnused(int step, int forceMemory = -1)
        {
            _clearList.Clear();
            var count = _cachedNames.Count;
            _lastPos = _lastPos >= count ? 0 : _lastPos;
            var end = _lastPos + step;
            for (var i = _lastPos; i < end && i < count; i++)
            {
                var assetName = _cachedNames[i];
                var cached = Find(assetName);
                if (cached == null)
                {
                    Debug.LogError($"[{nameof(CachedAssetContainer)}.{nameof(CleanUnused)}]" +
                                   $"Can't find asset:'{assetName}' of _cachedNames");
                    _clearList.Add(assetName);
                }
                else if (cached.TickGC(forceMemory))
                {
                    _clearList.Add(assetName);
                }
            }

            _lastPos = end;
            foreach (var item in _clearList)
            {
                Remove(item);
            }

            _clearList.Clear();
        }

        /// 强制清理所有缓存
        /// </summary>

            CleanUp();
        }

            _invertedDict.Clear();
            _cachedNames.Clear();
            _assetMap.Clear();
            AssetRecordMgr.Clear();
        }
```

```csharp
            Profiler.BeginSample("CleanStep");
```

```csharp
//**************************************************************************************
//Create By fred on 2019/04/28
//
//@Description 按价格排序的缓存池
//
//  1、缓存池中每个Key下能缓存多个Value，以栈的方式存储，每次获取时拿到的是最后放入的Value
//  2、缓存池中每个Key对应一个价格Price，Price大的优先驻留缓存池，即超过上限时剔除Price最小的Key
//**************************************************************************************

using System;
using System.Collections;
using System.Collections.Generic;
using System.Linq;

namespace Package.Cache
{
    public class PricedCachePool<TKey, TValue> : IEnumerable<KeyValuePair<TKey, Stack<TValue>>>, IEnumerable
    {
        #region protected 成员

        internal protected struct PricedKey
        {
            internal readonly TKey key;
            internal ulong price;

            internal PricedKey(TKey key, ulong price)
            {
                this.key = key;
                this.price = price;
            }
        }

        protected sealed class Compare : IComparer<PricedKey>
        {
            int IComparer<PricedKey>.Compare(PricedKey x, PricedKey y)
            {
                if (x.price < y.price) return -1;
                if (x.price > y.price) return 1;

                return Comparer<TKey>.Default.Compare(x.key, y.key);
            }
        }

        protected int _poolCapacity;
        protected readonly int _replicaLimit;
        protected readonly SortedDictionary<PricedKey, Stack<TValue>> _pool;
        protected readonly Dictionary<TKey, ulong> _prices;

        #endregion

        public Action<TKey, IEnumerable<TValue>, ulong> CacheEvict { get; set; }

        public int Count => _pool.Count;

        public int Capacity
        {
            get => _poolCapacity;
            set
            {
                if (value < 0)
                    throw new ArgumentException("Capacity should be positive!");

                UpdateCapacity(value);
            }
        }

        public PricedCachePool(int capacity, int replicaLimit)
        {
            _poolCapacity = capacity;
            _replicaLimit = replicaLimit;
            _pool = new SortedDictionary<PricedKey, Stack<TValue>>(new Compare());
            _prices = new Dictionary<TKey, ulong>();
        }

        /// <summary>
        /// 尝试将要缓存的值放入缓存池中（会进行缓存数量、价格等检查）
        /// </summary>
        /// <param name="key">要缓存的键</param>
        /// <param name="value">要缓存的一个值</param>
        /// <param name="newPrice">键对应的最新价格</param>
        /// <returns>是否真正将值放入了缓冲池</returns>
        public bool TryPut(TKey key, TValue value, ulong newPrice)
        {
            if (_poolCapacity == 0)
            {
                return false;
            }

            if (_prices.TryGetValue(key, out var oldPrice))
            {
                var pricedKey = new PricedKey(key, oldPrice);
                var values = _pool[pricedKey];

                if (oldPrice != newPrice)
                {
                    _prices[key] = newPrice;
                    _pool.Remove(pricedKey);

                    pricedKey.price = newPrice;
                    _pool.Add(pricedKey, values);
                }

                if (values.Count < _replicaLimit)
                {
                    values.Push(value);
                    return true;
                }

                return false;
            }


            var addKey = new PricedKey(key, newPrice);
            var addValue = new Stack<TValue>();
            addValue.Push(value);
            _pool.Add(addKey, addValue);
            _prices.Add(key, newPrice);

            return true;
        }

        /// <summary>
        /// 尝试从缓存池中拿走一个值
        /// </summary>
        /// <param name="key"></param>
        /// <param name="value"></param>
        /// <returns></returns>
        public bool TryTake(TKey key, out TValue value)
        {
            if (_prices.TryGetValue(key, out var price))
            {
                var pricedKey = new PricedKey(key, price);
                var values = _pool[pricedKey];

                if (values.Count > 0)
                {
                    value = values.Pop();
                    return true;
                }
            }

            value = default;
            return false;
        }

        public void Clear()
        {
            _pool.Clear();
            _prices.Clear();
        }

        public void UpdateCapacity(int newCapacity)
        {
            _poolCapacity = newCapacity;

            while (_pool.Count > _poolCapacity)
            {
                var min = _pool.First();
                var evictKey = min.Key;
                var evictValue = min.Value;

                _pool.Remove(evictKey);
                _prices.Remove(evictKey.key);
                CacheEvict?.Invoke(evictKey.key, evictValue, evictKey.price);
            }
        }

        #region enumerate

        public struct Enumerator : IEnumerator<KeyValuePair<TKey, Stack<TValue>>>
        {
            private SortedDictionary<PricedKey, Stack<TValue>>.Enumerator _enumerator;

            internal Enumerator(SortedDictionary<PricedKey, Stack<TValue>> dictionary)
            {
                _enumerator = dictionary.GetEnumerator();
            }

            public bool MoveNext()
            {
                return _enumerator.MoveNext();
            }

            internal void Reset()
            {
                throw new NotImplementedException();
                // _enumerator.Reset();
            }

            void IEnumerator.Reset()
            {
                throw new NotImplementedException();
                // _enumerator.Reset();
            }

            public KeyValuePair<TKey, Stack<TValue>> Current
            {
                get
                {
                    var item = _enumerator.Current;
                    return new KeyValuePair<TKey, Stack<TValue>>(item.Key.key, item.Value);
                }
            }

            object IEnumerator.Current => Current;

            public void Dispose()

                _enumerator.Dispose();
            }
        }

        IEnumerator<KeyValuePair<TKey, Stack<TValue>>> IEnumerable<KeyValuePair<TKey, Stack<TValue>>>.GetEnumerator()
```

```csharp

//Create By fred on 2019/04/28

//@Description Least accumulative cost cache

using System;
using System.Collections.Generic;

namespace Package.Cache
{
    public class AccCostCachePool<TKey, TValue>
    {
        #region protected 成员
        protected readonly struct CostKey
        {
            internal CostKey(TKey key, ulong cost)
            {
                this.key = key;
                this.cost = cost;
            }

            internal readonly TKey key;
            internal readonly ulong cost;
        }

        protected sealed class Compare : IComparer<CostKey>
        {
            int IComparer<CostKey>.Compare(CostKey x, CostKey y)
            {
                if (x.cost < y.cost) return -1;
                if (x.cost > y.cost) return 1;

                return 0;
            }
        }

        protected readonly int _poolCapacity;
        protected readonly int _replicaLimit;
        protected readonly SortedDictionary<CostKey, Stack<TValue>> _pool;

        protected readonly Dictionary<TKey, ulong> _accCosts; // 记录各key对应的累积cost值

        #endregion

        public Action<TKey, IEnumerable<TValue>> CacheEvict { get; set; }

        public int ValueCount => _pool.Count;

        public AccCostCachePool(int capacity, int replicaLimit)
        {
            _poolCapacity = capacity;
            _replicaLimit = replicaLimit;
            _pool = new SortedDictionary<CostKey, Stack<TValue>>(new Compare());
            _accCosts = new Dictionary<TKey, ulong>();
        }

        /// <summary>
        /// 更新数据的累积cost，并尝试加入缓存
        /// </summary>
        /// <param name="key">数据的键</param>
        /// <param name="value">数据的值</param>
        /// <param name="cost">数据本次增加的cost</param>
        /// <returns>返回值表示是否真正加入了缓存</returns>
        public bool TryPut(TKey key, TValue value, ulong cost)
        {
            ulong newAccCost;

            // 1.更新累积cost
            if (_accCosts.TryGetValue(key, out ulong accCost))
            {
                newAccCost = cost + accCost;
                _accCosts[key] = newAccCost;
            }
            else
            {
                newAccCost = cost;
                _accCosts.Add(key, newAccCost);
            }

            // 2.移除旧的缓存值
            var min = _pool.Keys.GetEnumerator().Current;
            Stack<TValue> values = null;
            if (accCost >= min.cost)
            {
                CostKey costKey = new CostKey(key, accCost);
                if (_pool.TryGetValue(costKey, out values))
                {
                    _pool.Remove(costKey);
                }
            }

            // 3.添加新的缓存值
            bool shouldAdd = false;
            bool shouldEvict = false;
            if (ValueCount < _poolCapacity || key.Equals(min.key))
            {
                shouldAdd = true;
            }
            else if (newAccCost > min.cost)
            {
                shouldAdd = true;
                // 如果数量达到上限，就要剔除最小的
                if (ValueCount >= _poolCapacity)
                {
                    shouldEvict = true;
                }
            }

            if (shouldAdd)
            {
                CostKey newKey = new CostKey(key, newAccCost);
                if (values == null)
                {
                    values = new Stack<TValue>();
                    values.Push(value);
                }
                if (values.Count < _replicaLimit)
                {
                    values.Push(value);
                }
                _pool.Add(newKey, values);
            }

            if (shouldEvict)
            {
                var minValue = _pool[min];
                _pool.Remove(min);
                // 事件回调放到最后，能最大限度地避其带来的副作用
                CacheEvict?.Invoke(min.key, minValue);
            }

            return shouldAdd;
        }

        /// <summary>
        /// 
        /// </summary>
        /// <param name="key"></param>
        /// <param name="value"></param>
        /// <returns></returns>
        public bool TryGet(TKey key, out TValue value)
        {
            if (_accCosts.TryGetValue(key, out ulong cost))
            {
                CostKey costKey = new CostKey(key, cost);
                if (_pool.TryGetValue(costKey, out Stack<TValue> values))
                {
                    if (values.Count > 0)
                    {
                        value = values.Pop();
                        return true;
                    }
                }
            }

            value = default;
            return false;
        }
    }
}
```

## A.9 `LruCacheStrategy.cs`（通用 LRU 策略（KJ 已有 LruCachePolicy，供对照），159 行）

```csharp
using System;
using System.Collections.Generic;
using Cysharp.Text;

namespace Package.Cache
{
    /// <summary>
    /// LRU（Least Recently Used）缓存策略
    /// 使用递增计数器跟踪访问顺序，当缓存满时淘汰最久未使用的项
    /// 使用LinkedList维护访问顺序，避免排序操作，实现O(1)的插入、访问和淘汰
    /// </summary>
    public class LruCacheStrategy<KeyT> : AbstractStrategy<KeyT>
    {
        // 只记录空闲对象的访问顺序
        private readonly LinkedList<KeyT> _idleAccessOrder = new();
        private readonly Dictionary<KeyT, LinkedListNode<KeyT>> _idleNodeMap = new();

        // 记录正在使用的对象（用于状态跟踪，不参与淘汰）
        private readonly HashSet<KeyT> _inUseObjects = new();

        protected override List<Type> NeedStatisticsList()
        {
            return new List<Type>();
        }

        protected override void Put(KeyT key)
        {
            // 对象归还到池中，从使用中移除，加入空闲队列
            _inUseObjects.Remove(key);
            MoveToFirstIdle(key);

            // 只检查空闲对象的容量限制
            CheckIdleCapacityAndEvict();
        }

        protected override void Take(KeyT key)
        {
            // 对象被取走，从空闲队列移除，加入使用中
            if (_idleNodeMap.TryGetValue(key, out var node))
            {
                _idleAccessOrder.Remove(node);
                _idleNodeMap.Remove(key);
            }

            _inUseObjects.Add(key);
        }

        private void MoveToFirstIdle(KeyT key)
        {
            // 如果已在空闲队列中，先移除
            if (_idleNodeMap.TryGetValue(key, out var existingNode))
            {
                _idleAccessOrder.Remove(existingNode);
            }

            // 添加到空闲队列头部
            var newNode = _idleAccessOrder.AddFirst(key);
            _idleNodeMap[key] = newNode;
        }

        private void CheckIdleCapacityAndEvict()
        {
            if (Capacity <= 0)
            {
                // 容量为0时，清空所有空闲对象
                while (_idleAccessOrder.Count > 0)
                {
                    EvictLastIdle();
                }

                return;
            }

            // 只当空闲对象数量超过容量时才淘汰
            while (_idleAccessOrder.Count > Capacity)
            {
                EvictLastIdle();
            }
        }

        private void EvictLastIdle()
        {
            var lastNode = _idleAccessOrder.Last;
            if (lastNode == null) return;

            var keyToEvict = lastNode.Value;
            _idleAccessOrder.RemoveLast();
            _idleNodeMap.Remove(keyToEvict);

            // 触发淘汰事件
            Eviction?.Invoke(keyToEvict);
        }

        public override void Destroy(KeyT key)
        {
            base.Destroy(key);

            // 从所有数据结构中移除
            _inUseObjects.Remove(key);
            if (!_idleNodeMap.TryGetValue(key, out var node)) return;
            _idleAccessOrder.Remove(node);
            _idleNodeMap.Remove(key);
        }

        public override void Clear()
        {
            _idleAccessOrder.Clear();
            _idleNodeMap.Clear();
            _inUseObjects.Clear();
        }

        public override string ToString()
        {
            using var sb = ZString.CreateStringBuilder();

            sb.Append("LRU Strategy: ");
            sb.Append(_inUseObjects.Count);
            sb.Append(" in-use, ");
            sb.Append(_idleAccessOrder.Count);
            sb.Append("/");
            sb.Append(Capacity);
            sb.Append(" idle");

            if (Capacity > 0)
            {
                var idleUsageRate = (float)_idleAccessOrder.Count / Capacity * 100f;
                sb.Append(" (");
                sb.Append(idleUsageRate.ToString("F1"));
                sb.Append("% idle capacity)");
            }

            if (_idleAccessOrder.Count > 0)
            {
                sb.AppendLine();
                sb.Append("  Most Recent Idle: ");
                sb.Append(_idleAccessOrder.First?.Value.ToString() ?? "None");

                if (_idleAccessOrder.Count > 1)
                {
                    sb.AppendLine();
                    sb.Append("  Least Recent Idle: ");
                    sb.Append(_idleAccessOrder.Last?.Value.ToString() ?? "None");
                }
            }

            return sb.ToString();
        }
    }


    /// <summary>
    /// LRU缓存策略，默认使用string作为key
    /// </summary>
    public class LruCacheStrategy : LruCacheStrategy<string>
    {

    }
}
```

```csharp
    /// FIFO缓存淘汰策略实现
    /// 按照先进先出原则移除最早加入的缓存项
    /// </summary>
    public class FIFOCacheStrategy<KeyT> : AbstractStrategy<KeyT>
    {
        private List<KeyT> _caches;

        public FIFOCacheStrategy()
        {
            _caches = new List<KeyT>();
        }


        public override void Clear()
        {
            // Log.Cache.Info("LRUCacheStrategy.Clear!");
            _caches.Clear();
        }

        protected override List<Type> NeedStatisticsList()
        {
            return new List<Type>()
            {
            };
        }

        protected override void Put(KeyT key)
        {
            _caches.Add(key);
        }

        public override void BeforeTake(KeyT key)
        {
            base.BeforeTake(key);

            if (_caches.Contains(key))
                return;

            //说明是新增，那么需要检查是否超过预期了
            var need = _caches.Count + 1;
            var offset = need - Capacity;
            offset = Math.Min(offset, Capacity);
            if (offset <= 0)
                return;

            for (var i = 0; i < offset; i++)
            {
                Eviction?.Invoke(_caches[i]);
            }

            _caches.RemoveRange(0, offset);
        }

        protected override void Take(KeyT key)
        {
            if (_caches.Remove(key)) return;
        }

        public override void Update(float elapsed = 0)
        {
            var count = _caches.Count;
            var offset = count - Capacity;

            if (offset <= 0)
                return;

            for (var i = 0; i < offset; i++)
            {
                Eviction?.Invoke(_caches[i]);
            }

            _caches.RemoveRange(0, offset);
        }

        public override string ToString()
        {
            var sb = new StringBuilder();
            foreach(var cacheName in _caches)
            {
                sb.Append($"{cacheName}\t");
            }
            return $"FIFOCacheStrategy: {sb.ToString()}";
        }
```

```csharp
using System.Collections.Generic;

namespace Framework.Cache
{
    public sealed class LruCachePolicy<TKey> : ICacheEvictionPolicy<TKey>
        where TKey : notnull
    {
        private readonly LinkedList<TKey> _order = new();
        private readonly Dictionary<TKey, LinkedListNode<TKey>> _nodes = new();

        public void Touch(TKey key)
        {
            if (_nodes.TryGetValue(key, out var node))
            {
                _order.Remove(node);
                _order.AddFirst(node);
                return;
            }

            var newNode = _order.AddFirst(key);
            _nodes[key] = newNode;
        }

        public void Remove(TKey key)
        {
            if (!_nodes.TryGetValue(key, out var node))
            {
                return;
            }

            _order.Remove(node);
            _nodes.Remove(key);
        }

        public bool TrySelectEvictionCandidate(out TKey key)
        {
            var last = _order.Last;
            if (last is null)
            {
                key = default!;
                return false;
            }

            key = last.Value;
            return true;
        }

        public void Clear()
        {
            _order.Clear();
            _nodes.Clear();
        }
    }
}
```

```csharp
using System;
using System.Collections.Generic;

namespace Framework.Cache
{
    public sealed class ResourceCache<TKey, TValue> : ICacheResContainer<TKey, TValue>
        where TKey : notnull
    {
        private readonly Func<TKey, TValue> _factory;
        private readonly Action<TValue>? _reset;
        private readonly Dictionary<TKey, TValue> _values = new();

        public ResourceCache(Func<TKey, TValue> factory, Action<TValue>? reset = null)
        {
            _factory = factory ?? throw new ArgumentNullException(nameof(factory));
            _reset = reset;
        }

        public bool TryGet(TKey key, out TValue value) => _values.TryGetValue(key, out value!);

        public TValue GetOrCreate(TKey key)
        {
            if (_values.TryGetValue(key, out var value))
            {
                return value;
            }

            value = _factory(key);
            _values[key] = value;
            return value;
        }

        public bool TryRemove(TKey key, out TValue value)
        {
            if (_values.TryGetValue(key, out value!))
            {
                _values.Remove(key);
                _reset?.Invoke(value);
                return true;
            }

            return false;
        }

        public void Clear()
        {
            if (_reset != null)
            {
                foreach (var value in _values.Values)
                {
                    _reset(value);
                }
            }

            _values.Clear();
        }
    }
}
```

> **现状结论（改造要点）：** 上述骨架提供了**容器**与**淘汰策略抽象**，但：
> 1. 没有 **owner 引用计数**（`CachedAsset` 的核心，治 A1）；
> 2. 没有 **内存代价 / `Memory` 字段**（治 A3 的压力分级需要）；
> 3. 没有 **淘汰驱动器**（每帧 `Update` 驱动 `ICacheEvictionPolicy` 选候选并 `Remove`，治 A3）；
> 4. `AssetRuntime` **完全没使用**这套骨架，自己写了 `_assetHandles`/`_ownedAssetHandles`（即 A2 根源）。
>
> → 改造不是「另写一套」，而是：在 `ResourceCache` 之上加 owner 引用计数 + 内存代价（对应 `CachedAsset`），实现一个 `AssetCacheMgr` 式驱动器每帧调用 `LruCachePolicy`（或 Priced/AccCost）选候选并 `TryRemove`，并让 `AssetRuntime` 的所有加载/实例化入口统一走它。
