## 🔴 CRITICAL — Entry.cs 不会编译

**文件：** `Assets/Scripts/Boot/Launcher/Entry.cs`

**三个问题：**

1. **`using Framework.Log` 和 `using Framework.RuntimeLog`** — `Launcher.asmdef` 的 references 为 `[UniTask, YooAsset, HybridCLR.Runtime, AssetShared]`，不包含 `Log` 和 `RuntimeLog`。这两个 using 导致编译器报错。

2. **`new BootUpdateRunner(startupSettings, view)`** — 构造函数已改为 `private BootUpdateRunner(BootBridge bridge)`，旧签名 `(BootStartupSettings, IBootStartupView)` 已删除。

3. **catch 块用了 `RuntimeLogManager`、`RuntimeLogEntry`、`GameLogLevel`** — 全是 `Framework.RuntimeLog` 和 `Framework.Log` 中的类型，AOT 侧不可访问。

**原因：** `Entry.cs` 从原 `Boot/` 目录移到 `Launcher/Data/` 下后内容未更新，仍保留了拆分前的代码——直接引用 `BootUpdateRunner` 和 Framework 日志类型。

**修复：** `Entry.cs` 应简化为委托给 `BootLoader`（已在 Launcher 目录下）：

```csharp
using Cysharp.Threading.Tasks;
using UnityEngine;

namespace Boot
{
    public class Entry : MonoBehaviour
    {
        [SerializeField] private BootStartupSettings startupSettings = new();
        [SerializeField] private MonoBehaviour startupView;
        private BootLoader _loader;
        private bool _isRunning;

        private void Awake()
        {
            DontDestroyOnLoad(gameObject);
            RunStartupAsync().Forget();
        }

        public void Repair()
        {
            if (_isRunning) return;
            RunStartupAsync().Forget();
        }

        private async UniTaskVoid RunStartupAsync()
        {
            _isRunning = true;
            var view = startupView as IBootStartupView;
            _loader?.Dispose();
            _loader = new BootLoader(startupSettings, view);
            try
            {
                await _loader.RunAsync();
            }
            catch (Exception e)
            {
                Debug.LogError($"[Entry] Startup failed: {e}");
                view?.SetStatus("Startup failed");
                view?.SetRepairVisible(true);
            }
            finally { _isRunning = false; }
        }

        private void OnDestroy() => _loader?.Dispose();
    }
}
```

注意：异常处理改为用 `Debug.LogError`（Unity Engine API，AOT 侧可用）而非 `RuntimeLogManager`。

## 🟠 HIGH

### 1. `BootStartupLog` 只写 `persistentDataPath`——Editor 路径不便于查找

**文件：** `BootStartupLog.cs:64`

```csharp
var dir = Path.Combine(Application.persistentDataPath, "Logs", "Runtime");
```

`Application.persistentDataPath` 在 Editor 中指向深层临时目录（如 `C:\Users\...\AppData\LocalLow\...`），不便于人工排查 AOT 阶段崩溃。

**建议：** 加 `#if UNITY_EDITOR` 分支，与原有 `BootRuntimeLogBootstrap` 保持一致：

```csharp
#if UNITY_EDITOR
    var projectRoot = Directory.GetParent(Application.dataPath)?.FullName ?? Application.dataPath;
    var dir = Path.Combine(projectRoot, "Logs", "Runtime");
#else
    var dir = Path.Combine(Application.persistentDataPath, "Logs", "Runtime");
#endif
```

## 🟡 MEDIUM

### 3. `BootLoader` 用 `WaitForCompletion()` 同步等待异步操作

**文件：** `BootLoader.cs:91`

```csharp
var operation = package.InitializePackageAsync(BuildOptions(config, packageName));
operation.WaitForCompletion();
```

在当前上下文（主线程 UniTask 调用）中安全，但 `WaitForCompletion()` 在非主线程或 WebGL 等平台不可用，且与原有 `BootUpdateRunner.InitializeAssetsAsync` 的 async 轮询模式不一致。

**建议：** 改为 async 轮询，保持与原始代码一致的防御性：

```csharp
var initOp = package.InitializePackageAsync(BuildOptions(config, packageName));
await initOp.ToUniTask();
if (initOp.Status != EOperationStatus.Succeeded)
    throw new InvalidOperationException($"[BootLoader] YooAsset package initialization failed: {initOp.Error}");
return package.InitializePackage;
```

### 4. `BootRemoteService.CustomUrlProvider` 静态可变字段——测试隔离风险

**文件：** `YooAssetStrategy/BootRemoteService.cs:13`

```csharp
public static System.Func<string, IReadOnlyList<string>> CustomUrlProvider;
```

测试代码 `HybridCLRBootTests.BootRemoteService_CustomUrlProviderOverrides()` 正确地用 `try/finally` 重置。但如果某个测试在 `try` 块前抛出，`CustomUrlProvider` 不会被重置，影响后续测试。

**当前实现可接受，后续可用 `[TearDown]` 加固。**

### 5. `BootStartupLogEntry` 有 public setter——API 不防误用

**文件：** `BootStartupLog.cs:15-19`

```csharp
public sealed class BootStartupLogEntry
{
    public DateTime TimeUtc { get; set; }   // ← setter 公开
    public BootStartupLogLevel Level { get; set; }
    public string Message { get; set; }

```

当前没有代码在 `.Add()` 之后修改条目，但 API 不做防御，未来可能被误改。

**建议：** 改为 `{ get; init; }` 在构造时一次性赋值。

### 6. `AssetRuntime.WrapFromExistingPackage` 冗余赋值

**文件：** `AssetRuntime.cs` `WrapFromExistingPackage()` 方法内

### 7. `BootUpdateRunner.ReplayEarlyLogs` 死代码 null 检查

**文件：** `BootUpdateRunner.cs:121`

## 🟢 LOW

### 8. `IsAssemblyLoaded` 每次 O(n) 遍历

**文件：** `BootLoader.cs:226-235`

```csharp
foreach (var assembly in AppDomain.CurrentDomain.GetAssemblies())
{
    if (string.Equals(assembly.GetName().Name, assemblyName, StringComparison.Ordinal))
        return true;

```

热更 DLL 只有 ~10 个，程序集总数 ~80 个，实际无性能影响。如果追求极致可用 `HashSet<string>` 预缓存。

### 9. `BootAssemblyEntry` 和 `BootMetadataEntry` 结构相同

两者字段完全一致（`assemblyName`, `fileName`, `resourcesPath`, `assetPath`）。这是有意为之的语义分离——AOT metadata 和热更 DLL 是两个不同域，可能在未来分化。不需要合并。

### 10. 缺少 `BootBuildinQueryService` 和 `BootDecryptionService`

设计文档指定了这两个 AOT 策略类，但实际未创建。YooAsset 的 `CreateDefaultSandboxFileSystemParameters` 会自动创建默认实现，不需要它们也能正常工作。不需要补充。
